# RHight FE 화면별 요구사항 정의서

- 문서 버전: v1.0
- 작성 기준: 현재 `src/router/index.js` 및 각 `views` 컴포넌트
- 작성일: 2026-02-19
- 범위: 프론트엔드 화면 기능 요구사항(화면 단위)

## 1. 공통 요구사항

### 1.1 인증/인가
- `meta.requiresAuth: true` 화면은 비로그인 시 `/login`으로 이동한다.
- `meta.requiresAdmin: true` 화면은 관리자(`role=admin`)만 접근 가능하다.
- 로그인 상태에서 `/login` 접근 시 `/`로 이동한다.

### 1.2 공통 레이아웃
- 로그인 화면 제외 모든 화면은 상단 헤더/좌측 사이드바 레이아웃을 사용한다.
- 상단 네비게이션은 메인/인사/근태/급여/성과/전자결재/KMS 이동을 제공한다.
- 로그아웃 기능, 공지사항 이동, 사원 검색 모달 진입을 제공한다.

### 1.3 공통 예외 처리
- 권한 없는 페이지 접근 시 기본 페이지(`/`) 또는 섹션 기본 페이지로 리다이렉트한다.
- 라우트 파라미터 누락/오류 시 안전한 기본 경로로 이동한다.

## 3. 화면 전환/리다이렉트 요구사항

- `/admin` 접근 시 `/admin/main`으로 이동한다.
- `/hr` 접근 시 `/hr/my`로 이동한다.
- `/hr/:pathMatch(.*)*` 접근 시 `/hr/my`로 이동한다.
- `/attendance` 접근 시 `/attendance/my`로 이동한다.
- `/salary` 접근 시 `/salary/my`로 이동한다.
- `/performance` 접근 시 `/performance/dashboard`로 이동한다.

## 5. 후속 상세화 필요 항목

- 각 화면별 필드 단위 유효성(필수값, 길이, 포맷) 상세 스펙
- API 계약(요청/응답 스키마, 에러코드) 명세
- 권한 세분화(읽기/작성/승인/관리) 정책
- 화면별 상태 전이 다이어그램(특히 전자결재/근태 승인)
